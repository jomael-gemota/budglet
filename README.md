Budglet is a zero-noise expense tracker that logs spending in seconds, computes a daily/monthly budget status in real time, and fires a blunt **Reality Check** at you every time you add an entry. No accounts. No syncing. No cloud. Just you and your spending.

---

## Features

- **Instant expense logging** — type an amount and a label, hit enter
- **Daily & monthly budget tracking** — see at a glance how much you've used and how much is left
- **Daily Insight** — a one-liner summary of today's spending patterns (rule-based or GPT-powered)
- **Reality Check** — a brutally honest status + actionable tip recalculated on every entry
- **Budget MMR** — a ranking system (inspired by competitive games) that scores your spending discipline across 90 days
- **Under-budget streak** — consecutive days under budget, shown with 🔥
- **Expense history** — browse past spending by day
- **Multi-language** — English, Tagalog, Bisaya
- **Fully offline** — all data lives in `localStorage`, no backend, no account required
- **PWA** — installable from the browser on any device
- **Android APK** — native Android wrapper via Capacitor

---

## Budget MMR Tiers

Your MMR is computed from the last 90 days of spending. Days with no logged expenses are skipped. Under-budget days gain points; over-budget days lose more.

| Tier | MMR Range | Meaning |
|------|-----------|---------|
| 🔴 **Gastos Mode** | 0 – 399 | Consistently blowing the budget |
| 🟠 **Tipid-Tipid Pa** | 400 – 699 | Making an effort, barely |
| 🟡 **Sinisikap** | 700 – 999 | Trying hard, but not quite there |
| 🟢 **Maayos Na** | 1000 – 1299 | Solid discipline, staying on track |
| 🩵 **Ipon Warrior** | 1300 – 1599 | Consistently under budget |
| 🟣 **Kuripot God** | 1600+ | Absolute budget mastery |

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) 18+
- `npm` (comes with Node)

### Install & run locally

```bash
git clone <repo-url>
cd budglet
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

---

## Building

### Web (PWA)

```bash
npm run build
```

Outputs a production-ready PWA to `dist/`. The service worker pre-caches the app shell so it works fully offline after the first load.

### Android APK

Requires [Android Studio](https://developer.android.com/studio) with the Android SDK installed.

```bash
# Set SDK paths (adjust if your installation differs)
export JAVA_HOME="/path/to/android-studio/jbr"
export ANDROID_HOME="$HOME/AppData/Local/Android/Sdk"   # Windows
# export ANDROID_HOME="$HOME/Android/Sdk"               # Linux/macOS

npm run build:apk
```

This runs `npm run build` → syncs to the Android project → compiles with Gradle → copies the APK to `public/budglet.apk`.

The APK is also linked directly in the app (onboarding screen and main view) so users on the web version can download it in one tap.

> **Note:** The generated APK is a debug build signed with the default Android debug key. For Play Store distribution, configure a release keystore in `android/app/build.gradle`.

## Settings

| Setting | Description |
|---------|-------------|
| **Daily budget** | Your target spend per day. Everything is measured against this. |
| **Currency** | Symbol prepended to all amounts (e.g. `₱`, `$`, `€`). |
| **Language** | Insight and Reality Check copy language: English, Tagalog, or Bisaya. |
| **OpenAI API key** | Optional. When provided, daily insights and reality checks are generated by GPT instead of rule-based logic. |
| **AI Insights** | Toggle GPT-powered copy on/off without removing the API key. |

All settings are stored locally in `localStorage` — nothing is sent anywhere unless you enable AI and provide an API key, in which case your spending summary is sent to the OpenAI API.

## Data & Privacy

Budglet stores everything in your browser's `localStorage` under the key `budglet-storage`. No data leaves your device unless you opt into AI insights, in which case a compact spending summary (no raw labels) is sent to the OpenAI API with your own key.
